Invoice Preference Tokens

Invoice Preference text fields support dynamic tokens — placeholders that are replaced with live invoice data when an invoice is viewed or exported. Tokens are stored verbatim in the database and expanded only at render time, so saving a preference never alters previously issued invoices.

Token format

Tokens use single curly braces: {invoice.total}, {biller.name}, etc. This is intentionally different from Blade {{ }} syntax and is safe to store in the database.

Supported fields

Tokens are expanded in the following Invoice Preference fields (Settings → Invoice Preferences):

  • Detail heading
  • Detail line
  • Payment method
  • Payment line 1 — name and value
  • Payment line 2 — name and value
Example

Storing Please pay {invoice.owing} by {invoice.due_date}. Reference: {invoice.number} in the Detail Line field will render as, for example, Please pay $150.00 by 30 May 2026. Reference: INV-0042 when the invoice is viewed.

Invoice tokens
TokenValue
{invoice.number}Invoice number / ID
{invoice.date}Invoice date (locale-formatted)
{invoice.due_date}Due date (locale-formatted, blank if not set)
{invoice.total}Total amount including tax, e.g. $1,234.56
{invoice.subtotal}Subtotal before tax
{invoice.tax}Total tax amount
{invoice.paid}Amount already paid
{invoice.owing}Amount still owing
{invoice.currency}Currency sign only, e.g. $
{invoice.currency_code}Currency code, e.g. USD
{invoice.note}Invoice note text
{invoice.payment_term}Payment term code, e.g. NET_30
{invoice.payment_term_label}Payment term label, e.g. Net 30 days
Biller tokens
TokenValue

{biller.email}Biller email address
{biller.phone}Biller phone number
{biller.address}Street address
{biller.city}City
{biller.state}State / region
{biller.zip}Postcode / ZIP code
{biller.country}Country
